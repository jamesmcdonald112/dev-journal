## **1. Install Commitlint (commit message validation)**

  

Commitlint checks that commit messages follow the conventional rules.

```bash
npm install -D @commitlint/cli @commitlint/config-conventional
```

Create:

  

**commitlint.config.mjs**

```mjs
export default {
  extends: ['@commitlint/config-conventional'],
};
```

Commitlint now enforces rules like:

- feat: …
    
- fix: …
    
- chore: …
    
- docs: …
    
- refactor: …
    

---

## **2. Install Commitizen (interactive commit tool)**

```bash
npm install -D commitizen
```

---

## **3. Add the conventional changelog adapter**

  

This makes your repo _Commitizen-friendly_.

```bash
npx commitizen init cz-conventional-changelog --save-dev --save-exact
```

This automatically adds to your **package.json**:

```json
"config": {
  "commitizen": {
    "path": "./node_modules/cz-conventional-changelog"
  }
}
```

Commitizen will now know which commit format to use.

---

## **4. Add the commit script**

  

Inside package.json → "scripts":

```json
"commit": "cz"
```

You already have this in your project.

---

## **5. How to use Commitizen**

  

Stage your changes:

```
git add .
```

Run the guided commit:

```
npm run commit
```

Commitizen will prompt for:

- commit **type** (feat, fix, chore, refactor…)
    
- commit **scope** (sentry, pages, hero-section…)
    
- short **subject**
    
- optional **details**
    
- breaking changes (if any)
    

  

This ensures every commit is 100% conventional.

---

## **6. How Commitlint + Commitizen work together**

|**Tool**|**Role**|
|---|---|
|**Commitizen**|Helps you _write_ a correct commit message|
|**Commitlint**|Ensures all commits _follow the rules_ (optional Husky hook later)|

Commitizen prevents mistakes _before_ committing.

Commitlint catches mistakes _after_ committing (if hooks are enabled).
