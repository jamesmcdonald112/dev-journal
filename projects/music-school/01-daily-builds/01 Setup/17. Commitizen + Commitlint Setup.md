## **1. Install Commitlint (commit message validation)**

  

Commitlint enforces Conventional Commits rules on all commit messages.

```
npm install -D @commitlint/cli @commitlint/config-conventional
```

Create **commitlint.config.mjs**:

```
export default {
  extends: ['@commitlint/config-conventional'],
};
```

This enforces rules like:

- feat: ...
    
- fix: ...
    
- chore: ...
    
- docs: ...
    
- refactor: ...
    

---

## **2. Install czg (interactive commit tool – modern replacement for Commitizen)**

  

czg replaces Commitizen and avoids deprecated dependencies.

```
npm install -D czg
```

❌ Do **not** install:

- commitizen
    
- cz-conventional-changelog
    

  

They are no longer needed.

---

## **3. Add the commit script**

  

Inside **package.json → scripts**:

```
"commit": "czg"
```

That’s it.

No extra config block required.

---

## **4. How to create a commit**

  

Stage your changes:

```
git add .
```

Run the interactive commit prompt:

```
npm run commit
```

czg will guide you through:

- commit **type** (feat, fix, docs, etc.)
    
- optional **scope**
    
- short **subject**
    
- optional **body**
    
- breaking change flag (if applicable)
    

  

It outputs a **fully valid Conventional Commit**.

---

## **5. How Commitlint + czg work together**

|**Tool**|**Purpose**|
|---|---|
|**czg**|Helps you _write_ a correct commit message|
|**Commitlint**|Enforces the rules (can be hooked with Husky)|

- czg prevents bad commits up front
    
- commitlint blocks invalid commits if someone bypasses the prompt
    

  

This is the cleanest setup.
