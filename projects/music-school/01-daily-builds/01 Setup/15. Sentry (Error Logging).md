# **1. Install**

```
npx astro add @sentry/astro
```

The CLI:

- Installs @sentry/astro
    
- Adds the integration to astro.config.mjs
    
- Creates no config files — you create these manually.
    

---

# **2. Correct File Structure (required to avoid warnings)**

  

Create **both** files in the project root:

```
sentry.client.config.js
sentry.server.config.js
```

These are the **only places** where DSN + SDK options should go.

Anything like project, org, authToken, dsn, **must NOT** be inside astro.config.mjs anymore.

---

# **3. Correct Client Config**

  

**sentry.client.config.js**

```json
import * as Sentry from "@sentry/astro";

Sentry.init({
  dsn: "<YOUR DSN>",
  sendDefaultPii: false, // safer GDPR default
  tracesSampleRate: 0,   // disable tracing unless needed
  replaysSessionSampleRate: 0,
  replaysOnErrorSampleRate: 0,
});
```

---

# **4. Correct Server Config**

  

**sentry.server.config.js**

```json
import * as Sentry from "@sentry/astro";

Sentry.init({
  dsn: "<YOUR DSN>",
  sendDefaultPii: false,
  tracesSampleRate: 0,
});
```

---

# **5. astro.config.mjs — Only NON-deprecated options allowed**

  

You should **NOT** pass project, org, authToken, dsn, release, environment in here.

  

The only safe thing to configure in 2025 is **source-map upload**:

```json
import { defineConfig } from "astro/config";
import sentry from "@sentry/astro";
import react from "@astrojs/react";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
  integrations: [
    react(),
    sentry({
      sourceMapsUploadOptions: {
        enabled: true,
      },
    }),
  ],
});
```

This removes the exact warning you’re getting.

# **6. Environment Variable Setup**

  

Create:

  

### **.env.sentry-build-plugin**

```json
SENTRY_AUTH_TOKEN=sntrys_XXXX
```

⚠️ **Never** commit this file.

  

The token is required only for source-map uploading during a production build.

---

# **7. How It Works (Behind the Scenes)**

|**File**|**What goes here**|**Notes**|
|---|---|---|
|**astro.config.mjs**|ONLY integration registration + source map settings|No DSN, org, project, authToken|
|**sentry.client.config.js**|Sentry.init for browser|Includes DSN|
|**sentry.server.config.js**|Sentry.init for server|Includes DSN|
|**.env.sentry-build-plugin**|Auth token used during build|Never commit|

This is exactly what the warning told you:

  

> “Configure the Sentry SDK in your sentry.client.config.js or sentry.server.config.js files.”

# **9. Recommended Production Settings (Optional)**

  

If you want releases + readable stack traces:

```json
SENTRY_RELEASE=1.0.0
SENTRY_ENVIRONMENT=production
```

Then add to both configs:

```json
release: process.env.SENTRY_RELEASE,
environment: process.env.SENTRY_ENVIRONMENT,
```
