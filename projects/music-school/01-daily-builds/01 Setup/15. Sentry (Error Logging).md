# **1. Install Sentry**

```
npx astro add @sentry/astro
```

This:

- installs @sentry/astro
    
- updates astro.config.mjs
    
- does **NOT** create config files (you create them manually)
    

---

# **2. Required File Structure**

  

Create **two files in the project root**:

```
sentry.client.config.js
sentry.server.config.js
```

These handle **all SDK configuration**.

  

Astro **must** have both:

|**File**|**Runs In**|**Purpose**|
|---|---|---|
|sentry.client.config.js|Browser|Captures frontend errors|
|sentry.server.config.js|Node server / SSR|Captures backend errors|

Astro has **two runtimes**, therefore two SDK initializations.

---

# **3. Client SDK (Browser)**

  

**File:** sentry.client.config.js

```json
import * as Sentry from "@sentry/astro";

Sentry.init({
  dsn: <your-dns>,

  sendDefaultPii: false,   // safer GDPR default
  tracesSampleRate: 0,     // disable tracing
  replaysSessionSampleRate: 0,
  replaysOnErrorSampleRate: 0,

  // You asked that these remain in the file:
  org: "james-mcdonald",
  project: <your-project>,
  authToken: process.env.SENTRY_AUTH_TOKEN,
});
```

# **4. Server SDK (SSR / API / Server Errors)**

  

**File:** sentry.server.config.js

```json
import * as Sentry from "@sentry/astro";

Sentry.init({
  dsn: <your-dns>,

  sendDefaultPii: false,

```

# **5. astro.config.mjs — Minimal (No Deprecated Options)**

  

Only enable source-map uploading (optional but recommended):

```json
import { defineConfig } from "astro/config";
import tailwindcss from "@tailwindcss/vite";
import react from "@astrojs/react";
import sentry from "@sentry/astro";

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
  integrations: [
    react(),
    sentry(),
  ],
});
```

**NO DSN here.**

**NO org/project here.**

**NO authToken here.**

**NO tracing settings here.**

  

Those go in the client/server config files only.

# **6. Auth Token File**

  

Sentry requires an auth token for uploading sourcemaps during production builds.

  

Create:

  

### **.env.sentry-build-plugin**

```
SENTRY_AUTH_TOKEN=sntrys_xxxxxxxxxxxxxxxxxxxx
```

Never commit this file.

---

# **7. Why Sentry Has Client + Server Files**

  

Astro runs in **two completely different environments**:

  

### **Client-side Sentry**

- catches UI errors
    
- catches click handlers
    
- catches hydration errors
    
- catches network errors
    
- shows toast-level issues
    

  

### **Server-side Sentry**

- catches SSR crashes
    
- catches API route errors
    
- catches middleware errors
    
- captures request metadata
    
- logs backend exceptions
    

  

They **must** be initialized separately, otherwise half your errors go missing.

```

  Throw Test Error
</button>
```

Run:

```
npm run dev
```

# **9. Production Enhancements (Optional)**

  

Add to .env:

```
SENTRY_ENVIRONMENT=production
SENTRY_RELEASE=1.0.0
```

Then in both configs:

```
release: process.env.SENTRY_RELEASE,
environment: process.env.SENTRY_ENVIRONMENT,
```
