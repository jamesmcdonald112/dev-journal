# **Obsidian Note — Sentry + Astro (Clean Version)**

  

## **1. Install & Setup**

```bash
npx astro add @sentry/astro
```

Create:

```
sentry.client.config.js
sentry.server.config.js
```

Each file should contain:

```json
import * as Sentry from "@sentry/astro";

Sentry.init({
  dsn: "<DSN from Sentry project>",
  sendDefaultPii: true,
});
```

Add to .env:

```
SENTRY_AUTH_TOKEN=sntrys_...
```

Add integration in astro.config.mjs:

```json
integrations: [
  sentry({
    project: "<project-slug>",
    org: "<org-slug>",
    authToken: process.env.SENTRY_AUTH_TOKEN,
  }),
]
```

---

## **2. Development vs Production**

  

### **✔ Development**

- Sentry captures errors normally.
    
- No release tracking.
    
- No source maps.
    
- Do **not** set environment or release yet.
    

  

### **✔ Production (deployment)**

  

Add:

```json
environment: "production",
release: process.env.npm_package_version,
```

Sentry will:

- upload source maps
    
- create a release
    
- show readable stack traces
    
- detect regressions

## **4. When you deploy**

  

Hosting platforms (Netlify/Vercel/Cloudflare) automatically run:

```bash
npm install
npm run build
```

and upload your build.

  

Don’t commit dist/.
