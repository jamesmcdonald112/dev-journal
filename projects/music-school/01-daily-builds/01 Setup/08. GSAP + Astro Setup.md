## **1. Install GSAP**

  

Run:

```bash
npm install gsap
```

This makes GSAP available inside any .astro component.

---

## **2. Use GSAP inside a `<script>` tag in a .astro file**

  

Astro allows client-side scripts directly in components.

  

**Important:**
    
- GSAP must run in the browser (client), not during server-side rendering.
    

  

Example component:

  

### **src/components/TestGsap.astro**

```ts
---
---
<div id="box" class="w-32 h-32 bg-green-400"></div>

<script>
  import { gsap } from "gsap";

  gsap.from("#box", {
    opacity: 0,
    y: -50,
    duration: 0.8,
  });
</script>
```

This animates the green box when the page loads.

---

## **3. Use your GSAP component in a page**

  

### **src/pages/index.astro**

```ts
---
import TestGsap from "../components/TestGsap.astro";
import Layout from "../layouts/Layout.astro";
---

<Layout 
  title="Harmony Music Academy"
  description="Dublin’s premier music school"
>
  <h1 class="text-4xl font-bold text-green-400">Hello g</h1>

  <TestGsap />
</Layout>
```

# **4. When to Use GSAP (✔ Use GSAP for these)**

  

GSAP is only needed when Tailwind/CSS cannot do the job:

  

### **✔ Scroll-based animations**

- Parallax
    
- Reveal-on-scroll
    
- Sticky/pinned sections
    
- Scroll timelines
    

  

### **✔ Timeline sequences**

- Animation A → B → C with delays
    
- Coordinating multiple elements
    

  

### **✔ Stagger effects**

- Animate 10+ elements in a wave
    

  

### **✔ Complex hero animations**

- Logo + text + images moving together
    
- High-end landing-page effects
    

  

### **✔ SVG animations**

- Morphing
    
- Drawing
    
- Motion paths
    
- Split text
    

  

### **✔ Physics + spring motion**

- Elastic
    
- Bounce
    
- Natural motion
    

  

### **✔ Advanced interactive menus**

- Nav open/close animations
    
- Smooth transitions
    
- Reversible timelines
    

---

# **5. What NOT to use GSAP for (Use Tailwind instead)**

  

Tailwind is:

- faster
    
- zero JavaScript
    
- perfect for Astro
    

  

Use Tailwind for:

  

### **✔ Simple fades**

  

### **✔ Slides + transforms**

  

### **✔ Hover animations**

  

### **✔ Buttons / cards**

  

### **✔ Simple modals**

  

### **✔ Accordions / dropdowns (with small Astro scripts)**

  

### **✔ Page transitions (Astro View Transitions)**

---

# **6. Final Note**

  

### **Tailwind = default choice for most animations**

- No JS
    
- No hydration
    
- Best performance for Astro
    

  

### **GSAP = only for advanced animation where Tailwind is not enough**

- Use sparingly
    
- Only where you need timelines, scroll effects, or complex movement
