
# Docs used: [GitHub](https://github.com/tailwindlabs/tailwindcss-typography)

A plugin that gives you **beautiful default typography styles** for HTML content you _don‚Äôt fully control_, such as:

- Markdown content
    
- Blog articles
    
- CMS content
    
- Long text pages (Terms, Privacy, About Us)
    
- Lesson descriptions / teacher bios
    
- Rich text in a blog
    

  

It applies a **‚Äúprose‚Äù design system** that makes paragraphs, headings, links, lists, tables, blockquotes, images, etc. look automatically good ‚Äî without writing custom CSS.

# **‚úÖ 1. Install the plugin**

# **‚úÖ 2. Register the plugin (Tailwind v4 way)**

  

In **src/styles/global.css** (or wherever you imported Tailwind):


‚úî No Tailwind config file needed in v4

‚úî This is the official recommended method

---

# **‚úÖ 3. Basic Usage**

  

Using typography = wrap your content in a .prose class.

  

Example:

```html
<article class="prose">
  <h1>How to choose the right music teacher</h1>
  <p>
    Some text here explaining the process of selecting a teacher...
  </p>
  <h2>Key things to look for</h2>
  <ul>
    <li>Experience</li>
    <li>Communication</li>
    <li>Qualifications</li>
  </ul>
</article>
```

‚úî Automatically styles text

‚úî Perfect spacing

‚úî Readable line height

‚úî Fixes images, lists, tables, blockquotes

---

# **‚úÖ 4. Optional: Adjust size using modifiers**

  

Example:

```
<article class="prose prose-lg">
  {{ markdown }}
</article>
```

Available sizes:

|**Class**|**Body size**|
|---|---|
|prose-sm|14px|
|prose-base|16px|
|prose-lg|18px|
|prose-xl|20px|
|prose-2xl|24px|

---

# **‚úÖ 5. Optional: Different colour themes**

  

Example:

```html
<article class="prose prose-slate">
  ...
</article>
```

Available themes:

- prose-gray (default)
    
- prose-slate
    
- prose-zinc
    
- prose-neutral
    
- prose-stone
    

---

# **‚úÖ 6. Optional: Dark mode**

```html
<article class="prose dark:prose-invert">
```

This automatically inverts headings, text, code blocks, lists, etc.

---

# **‚úÖ 7. Element-level modifications**

  

This is powerful.

  

Example:

```html
<article class="prose prose-a:text-green-600 prose-img:rounded-xl">
  ...
</article>
```

You can target:

- prose-headings:*
    
- prose-a:*
    
- prose-img:*
    
- prose-code:*
    
- prose-h1:*
    
- prose-p:*
    
- prose-li:*
    
- etc.
    

  

Example: underline all links:

```html
<article class="prose prose-a:underline">
```

---

# **‚úÖ 8. Removing prose for a specific block**

  

Use not-prose:

```html
<article class="prose">
  <h1>Title</h1>

  <div class="not-prose">
    <button class="bg-green-600 text-white px-4 py-2 rounded">Normal Button</button>
  </div>

  <p>More content...</p>
</article>
```

- Music theory articles
    
- Teacher biography pages
    
- ‚ÄúAbout Us‚Äù long text
    
- Privacy policy and terms pages
    
- Pricing descriptions
    
- Lessons explanation pages
    
- Any long-form text content
    

  

You **won‚Äôt** use it for:

- Navigation
    
- Hero sections
    
- Cards
    
- Buttons
    
- General UI
    

  

Typography is for **rich text content**, not UI layout.
