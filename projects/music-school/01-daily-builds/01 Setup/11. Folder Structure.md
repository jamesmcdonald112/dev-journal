---

# **1. Core Folders**

```
src/
  pages/       → Route files (.astro)
  layouts/     → Base layouts (SiteLayout, MarketingLayout)
  styles/      → Global CSS, Tailwind imports
  assets/      → Images, icons, static files
  lib/         → Utilities, types, helpers
  content/     → Content collections (Zod schemas later)
```

---

# **2. Components**

```
src/components/
  ui/          → Reusable UI primitives (Button, Card, Input)
  shared/      → Site-wide components (Navbar, Footer)
```

UI components = tiny building blocks

Shared components = bigger, global components

---

# **3. Sections (Page-Level Sections)**

  

Sections are isolated “chunks” used to build pages.

```
src/sections/
  hero/
    Hero.astro

  pricing/
    Pricing.astro

  testimonials/
    Testimonials.astro

  faq/
    FAQ.astro

  cta/
    CTA.astro
```

Each section has its own folder so you can add:

- images
    
- local helpers
    
- scripts
    
- variants
    

  

without polluting other folders.

---

# **4. Features (Logic + UI Together)**

  

For complex areas that require:

- form handling
    
- validation
    
- loaders
    
- local state
    
- data fetching
    
- Zod schemas
    

```
src/features/
  contact-form/
    ContactForm.astro
    contact.schema.ts
    submit.ts

  courses/
    CourseList.astro
    course.data.ts
```

Features are self-contained and scalable.

---

# **5. Utility Files**

```
src/lib/utils.ts   → Shared helpers
src/lib/types.ts   → Global TypeScript types
```

---

# **Commands Used**

  

Paste this to recreate the full structure:

```
mkdir -p src/{pages,layouts,styles,assets,lib,content}
mkdir -p src/components/{ui,shared}
mkdir -p src/sections/{hero,pricing,testimonials,faq,cta}
mkdir -p src/features/{contact-form,courses}

touch src/components/ui/Button.astro
touch src/components/shared/Navbar.astro
touch src/components/shared/Footer.astro

touch src/sections/hero/Hero.astro
touch src/sections/pricing/Pricing.astro

touch src/sections/faq/FAQ.astro
```
