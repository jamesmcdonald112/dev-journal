## **1. Install the plugin**

```
npm i -D prettier prettier-plugin-tailwindcss
```

---

## **2. Add a .prettierrc**

  

Use exactly this (nothing more):

```
{
  "plugins": ["prettier-plugin-tailwindcss"],
  "tailwindStylesheet": "./src/styles/global.css"
}
```

- plugins → loads the Tailwind sorter
    
- tailwindStylesheet → required for Tailwind v4 so it knows your classes
    

---

## **3. VS Code settings**

  

Biome = default formatter

Prettier = only for CSS + Astro

  

Your settings.json should include:

```
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "biomejs.biome",

  "[css]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[astro]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

That’s it.

No other settings are required.

---

## **4. How to verify it works**

  

Open any .astro or .css file → change a class order → save.

  

Example:

```
<div class="text-white px-4 bg-red-500 flex mt-5"></div>
```

It should auto-sort to:

```
<div class="mt-5 flex bg-red-500 px-4 text-white"></div>
```

If that happens, everything is configured correctly.

---
## 5. Exclude global.css from Biome

Biome cannot format Tailwind v4 CSS correctly yet.

Add this to biome.json so Biome skips global.css:

```json
{
  "files": {
    "includes": ["**", "!**/global.css"]
  }
}
```

Prettier will format global.css instead.
