## **How styling works in this project**

  

### **1. Design tokens (source of truth)**

  

Defined using Tailwind @theme and **named following Shadcn conventions**:

- --color-primary
    
- --color-primary-foreground
    
- --color-muted
    
- --color-muted-foreground
    
- --radius
    

  

These tokens:

- Represent **meaning**, not raw values
    
- Automatically generate Tailwind utilities
    
    (e.g. bg-primary, text-muted-foreground)

Tailwind Theme Variables

https://tailwindcss.com/docs/theme

### **2. Utilities (how styles are applied)**

  

UI is styled primarily using **Tailwind utility classes**:

- bg-primary
    
- text-foreground
    
- hover:bg-primary-hover
    
- ring-primary
    

  

Utilities are **not custom CSS** â€” they are generated from theme tokens.

  

ðŸ“– Reference (core mental model):

https://tailwindcss.com/docs/styling-with-utility-classes

---

### **3. Component classes (when utilities get repetitive)**

  

For repeated patterns (buttons, inputs, form fields), we use:

```
@layer components {
  .btn-primary { ... }
}
```

Rules:

- Components may use @apply
    
- Components must only reference **theme tokens**, never raw colors
    
- Utilities can still override component styles if needed
