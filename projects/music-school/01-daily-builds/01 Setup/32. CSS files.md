I use **CUBE CSS** to **organise CSS**, **Tailwind to implement styles**, **Every Layout for layout patterns**, and **GOV.UK Design System for accessibility patterns**.

  

CUBE is **structure**, not a framework.

---

## **What CUBE means (plain English)**

  

**CUBE = Composition · Utility · Block · Exception**

  

It answers one question only:

  

> _Where does this CSS belong?_

---

## **How this maps to my project**

  

### **File structure**

```
src/styles/
  global.css          → entry point (imports everything once)
  reset.css           → small overrides only (Tailwind Preflight is the base)
  tokens.css          → design values (colors, spacing, radius)
  composition.css     → layout & spacing patterns
  utilities.css       → tiny generic helpers
  blocks/
    buttons.css
    forms.css
    cards.css
```

---

## **1. CSS Reset**

  

**What it is**

- The baseline browser reset
    

  

**What I use**

- **Tailwind Preflight** (this replaces normalize / Meyer / Andy Bell resets)
    

  

**What I add on top (optional)**

- body { min-height: 100vh }
    
- reduced-motion hardening
    
- optional smooth scroll
    

  

**Rule**

  

> Do NOT include full resets when using Tailwind.

## **2. Composition**

  

**What it is**

- Controls **layout and spacing between elements**
    
- The **skeleton** of pages and components
    

  

**What belongs here**

- Page structure
    
- Content flow / rhythm
    
- Reusable layout patterns
    

  

**Examples**

- .flow → vertical spacing between children
    
- .stack → vertical layout with gap
    
- .cluster → inline items that wrap
    
- .wrapper → centered page container
    

  

**Source**

- **Every Layout** (book)
    

  

➡ Details:

[[css/best-practices/2. Composition|2. Composition]]

## **3. Utility Classes**

  

**What they are**

- **Small, generic helpers**
    
- Do **one job**
    
- Used anywhere
    

  

**Examples**

- .list-reset
    
- .sr-only (if custom)
    
- .text-center
    

  

**Tailwind note**

- Tailwind already provides most utilities
    
- Only add custom ones when needed
    

  

➡ Details:

[[css/best-practices/3. Utility Classes|3. Utility Classes]]

---

## **4. Blocks**

  

**What they are**

- **Actual components**
    
- Where most styling lives
    

  

**Examples**

- .btn, .btn-primary
    
- .form-input
    
- .card
    

  

**Mapping**

- React / Astro component = structure
    
- Block CSS = visual + interaction styling
    

  

**Implementation**

- @layer components
    
- Uses @apply
    

  

➡ Details:

[[css/best-practices/4. Block|4. Block]]

---

## **5. Exceptions**

  

**What they are**

- **Rare variations of a block**
    
- Usually state-based
    

  

**Example**

```
<article class="card" data-state="reversed"></article>
```

**Rule**

- Live next to the block
    
- If you need many → create a new block
    

  

➡ Details:

[[css/best-practices/5. Exceptions|5. Exceptions]]

---

## **6. Tokens**

  

**What they are**

- **Named design values**
    
- No layout, no components
    

  

**Examples**

- Colors
    
- Spacing
    
- Font sizes
    
- Radius
    

  

**Where they live**

- tokens.css using @theme
    

  

**Why**

- Easy theming
    
- Consistency across sites
    

  

➡ Details:

[[css/best-practices/6. Tokens|6. Tokens]]
