Biome currently has **experimental** and **partial** support for .astro files, and causes issues with formatting, false lint warnings, and unused-prop errors.

So this project uses **Prettier only** for consistent formatting.

---

# **1. Install Prettier + Astro plugin**

```bash
npm install --save-dev --save-exact prettier prettier-plugin-astro prettier-plugin-tailwindcss
```

---

# **2. Add a .prettierrc config**

  

Create .prettierrc in the root:

```json
{
  "plugins": [
    "prettier-plugin-astro",
    "prettier-plugin-tailwindcss"
  ],
  "overrides": [
    {
      "files": "*.astro",
      "options": {
        "parser": "astro"
      }
    }
  ]
}
```

> **Note:** Tailwind plugin must always be last.

---

# **3. VS Code settings**

  

Add to .vscode/settings.json:

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[astro]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

This guarantees that .astro files format correctly on save.

---

# **4. Add a format script**

  

In package.json:

```json
"scripts": {
  "dev": "astro dev",
  "build": "astro build",
  "preview": "astro preview",
  "format": "prettier --write ."
}
```

---

# **5. Format your entire project**

```
npm run format
```

This will format:

- .astro
    
- .js / .ts
    
- .css
    
- Inline scripts/templates
    
- Tailwind class sorting
