# **✅** 

# **Step 1 — Add Tailwind using Astro’s built-in command**

  

Run this inside your project folder:

```bash
npx astro add tailwind
```

Choose **Yes** for all prompts:

- Install Tailwind? → **Yes**
    
- Create global.css? → **Yes**
    
- Update astro.config.mjs? → **Yes**
    

  

Astro automatically performs:

  

### **✔ Installs:**

- tailwindcss
    
- @tailwindcss/vite
    

  

### Updates your

### **astro.config.mjs**

### **:**

```ts
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  vite: {
    plugins: [tailwindcss()]
  }
});
```

### **✔ Creates the stylesheet:**

```
src/styles/global.css
```

with:

```css
@import "tailwindcss";
```

This loads Tailwind’s base layers.

---

# **✅** 

# **Step 2 — Create your layout file and import Tailwind**

  

Astro requires **you** to create your own layout.

This is where global Tailwind styling is imported.

  

Create:

```
src/layouts/Layout.astro
```

Add this exact content:

```ts
---
import '../styles/global.css';

interface Props {
  title: string;
  description: string;
}

const { title, description } = Astro.props;
---

<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="description" content={description} />
    <title>{title}</title>
  </head>
  <body class="antialiased">
    <slot />
  </body>
</html>
```

This layout:

- Imports Tailwind globally
    
- Sets the HTML document shell
    
- Handles title + description
    
- Wraps page content using <slot />
    

  

(From docs: https://docs.astro.build/en/basics/layouts/)

---

# **✅** 

# **Step 3 — Wrap your pages with the layout**

  

Here’s your actual working index.astro:

```ts
---
import Layout from '../layouts/Layout.astro';
---

<Layout 
  title="Harmony Music Academy"
  description="Dublin’s premier music school"
>
  <h1 class="text-4xl font-bold text-green-400">
    Hello g
  </h1>
</Layout>
```

This ensures:

- The entire page is inside your layout
    
- Tailwind classes work
    
- Title and description appear in `<head>`
    
- `<slot />` receives your content

# Tailwindcss-animate in Astro

## Offical Docs - [github](https://github.com/jamiebuilds/tailwindcss-animate)
  

## **✅** 

## **1. Install the plugin**

  

Run:

```bash
npm install -D tailwindcss-animate
```

---

## **✅** 

## **2. Enable it in Astro (Tailwind v4 syntax)**

  

Because Astro uses **CSS-based Tailwind config**, you add the plugin in your **global stylesheet**:

  

### **src/styles/global.css**

###  **(or whatever your main file is)**

```css
@import "tailwindcss";
@plugin "tailwindcss-animate";
```

No JS config.

No tailwind.config.js.

---

## **✅** 

## **3. Usage Examples**

  

### **Fade-in**

```html
<div class="animate-in fade-in duration-500">
  Hello
</div>
```

## **❌ NOT for complex motion**

  

Do **not** use tailwindcss-animate for:

- chained animations
    
- complex sequences
    
- scroll-triggered animations
    
- staggered timelines
    
- physics / fluid motion
    

  

→ Use **GSAP** for that.

# Tailwindcss/forms in Astro (Tailwind v4)

## Docs - [GitHub](https://github.com/tailwindlabs/tailwindcss-forms?tab=readme-ov-file)

## **✅** 

## **1. Install the plugin**

  

Run:

```bash
npm install -D @tailwindcss/forms
```

---

## **✅** 

## **2. Enable it in Astro (Tailwind v4 syntax)**

  

Astro uses **CSS-based Tailwind setup**.

  

So you register the plugin _inside your global stylesheet_.

  

### **Example:** 

### **src/styles/global.css**

```
@import "tailwindcss";
@plugin "@tailwindcss/forms";
```

✔ No tailwind.config.js needed

✔ No JS setup

✔ Astro loads it automatically

---

## **✅** 

## **3. What the plugin actually does**

  

It provides a **basic form reset** so that form elements become:

- visually consistent across browsers
    
- easy to style using Tailwind utilities
    
- easy to customise without writing custom CSS
    

  

It normalises these elements:

- all input[type=...]
    
- select, multiple selects
    
- checkbox
    
- radio
    
- textarea
    

---
  

### **✔ After plugin (utility-friendly reset)**

  

Now you can do:

```html
<select class="rounded-full px-4 py-3"></select>
```

✔ Fully styleable

✔ No custom CSS

✔ Works the same across browsers

  

Checkbox example:

``` html
<input type="checkbox" class="rounded text-pink-500" />
```

✔ Tailwind utility applies

✔ Even checkbox accepts colors

---

## **✅** 

## **5. Using plugin-provided classes (form-*)**

  

You can also explicitly apply the form styles to non-form elements.

  

Examples:

```html
<input type="email" class="form-input rounded-full px-4 py-3" />

<select class="form-select rounded-full px-4 py-3"></select>

<input type="checkbox" class="form-checkbox rounded text-green-500" />
```

---

## **🔍** 

## **6. Strategy options**

  

(You probably won’t use these, but included for completeness)

  

You can decide if the plugin should:

  

### **A. Apply global form resets (default)**

```json
@plugin "@tailwindcss/forms" {
  strategy: "base";
}
```

### **B. Only generate classes (form-input)**

```json
@plugin "@tailwindcss/forms" {
  strategy: "class";
}
```

Default = **global styles + classes**

Keep default — it’s what you want.
