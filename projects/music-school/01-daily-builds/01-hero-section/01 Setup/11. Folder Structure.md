You created a scalable Astro folder structure based on best practices from Astro + “Bulletproof React” architecture.

  

### **Created core folders**

```
src/
  pages/          → Actual route files (.astro)
  layouts/        → Base layouts (BaseLayout, MarketingLayout, etc.)
  styles/         → Global CSS, variables, theme
  assets/         → Images, icons, SVGs, static assets
  lib/            → Utilities, types, helper functions
  content/        → Content collections (Zod schemas later)
```

---

### **Components**

```
src/components/
  ui/             → Reusable UI primitives (Button, Card, Input)
  shared/         → Site-wide components (Navbar, Footer)
```

---

### **Sections (Page Sections)**

  

Used to build pages from isolated blocks.

```
src/sections/
  hero/
  pricing/
  testimonials/
  faq/
  cta/
  features/
```

Each section gets its own folder + .astro file so it can have its own images, styles, or helpers.

---

### **Features (Logic-heavy areas)**

  

Used when UI + logic belong together (forms, data, dynamic content).

```
src/features/
  contact-form/
    ContactForm.astro
    contact.schema.ts
    submit.ts

  courses/
    CourseList.astro
    course.data.ts
```

Features can contain:

- components
    
- API logic
    
- schemas (Zod)
    
- data loaders
    
- utils specific to that feature
    

  

This prevents mixing logic with simple UI/sections.

---

### **Utility Files**

```
src/lib/utils.ts      → helpers shared across project
src/lib/types.ts      → global TypeScript types
```

# **✔ Summary**

- **Components** → generic building blocks
    
- **Sections** → page sections (Hero, Pricing, etc.)
    
- **Features** → logic-heavy modules (forms, courses, authentication later)
    
- **Layouts** → global wrappers
    
- **Pages** → routes
    
- **Content** → Zod + content collections (for later)
    

  

This structure gives you:

- clean separation of concerns
    
- scalable organisation
    
- easier testing
    
- predictable imports
    
- matches modern Astro best practices
