### **1) Install Tailwind normally (Astro integration or Tailwind setup)**

  

Goal: Tailwind utilities compile and are available in .astro templates.

  

### **2) Create a single global stylesheet and import Tailwind first**

  

src/styles/global.css:

```
@import "tailwindcss";
@import "./reset.css";
@import "./tokens.css";
```

**Why this order**

- Tailwind first so utilities and base layers exist.
    
- Reset next to standardise browser defaults.
    
- Tokens last so our theme values are defined and available to utilities.
    

  

### **3) Import** 

### **global.css**

###  **once in the base layout**

  

This guarantees every page uses the same theme and reset.

  

### **4) Define theme tokens in** 

### **tokens.css**

  

src/styles/tokens.css:

```
@theme {
  --color-bg: var(--color-slate-50);

  --color-surface: var(--color-white);
  --color-surfaceHover: var(--color-slate-50);
  --color-surfaceRaised: var(--color-slate-100);

  --color-text: var(--color-slate-900);
  --color-muted: var(--color-slate-600);
  --color-border: var(--color-slate-200);

  --color-textOnDark: var(--color-white);
  --color-mutedOnDark: var(--color-slate-200);

  --color-accent: var(--color-slate-900);
  --color-accentHover: var(--color-slate-700);
  --color-accentContrast: var(--color-white);
}
```

### **5) Apply base theme on** 

### **`<body>`**

  

In layout:

```
<body class="bg-bg text-text">
  <main><slot /></main>
</body>
```

**Why**

- Sets the default “canvas” background and base text once.
    
- Everything else inherits, so you don’t repeat colours.
