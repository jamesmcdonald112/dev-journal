# **🔷 Overview – What We’re Building**

  

Form fields:

- name (required)
    
- phone (required)
    
- email (required) ← strongly recommended
    
- practiceArea (required dropdown)
    
- message (optional)
    
- website (hidden honeypot)
    

  

Architecture:

```
Form (UI)
   ↓
Zod schema (runtime validation)
   ↓
Astro Action (security + branching)
   ↓
Service layer (send email)
   ↓
Resend
```

Security layers:

- Zod runtime validation
    
- Honeypot
    
- Origin allowlist
    
- Header injection protection
    
- HTML escaping
    
- Env-based configuration
    
- Tests

# **3️⃣ Zod Schema (Runtime Validation + Types)**

  

**File:** contact.schema.ts

```
import { z } from "zod";
import { PRACTICE_AREAS } from "./config/practice-areas";

export const contactSchema = z.object({
  name: z.string()
    .trim()
    .min(2, "Name is required.")
    .max(100, "Name is too long."),

  phone: z.string()
    .trim()
    .min(5, "Phone number is required.")
    .max(30, "Phone number is too long."),

  email: z.string()
    .trim()
    .email("Enter a valid email address.")
    .max(254),

  practiceArea: z.enum(PRACTICE_AREAS, {
    errorMap: () => ({ message: "Please select a practice area." }),
  }),

  message: z.string()
    .trim()
    .max(2000, "Message is too long.")
    .optional(),

  // Honeypot
  website: z.string()
    .trim()
    .nullable()
    .optional()
    .transform((v) => v ?? ""),
});

export type ContactFormData = z.infer<typeof contactSchema>;
```

What this gives you:

- Runtime validation
    
- Clean trimmed data
    
- Always-string honeypot
    
- Enum safety
    
- Compile-time inference

# **4️⃣ Astro Action (Security + Branching Only)**

  

**File:** actions/sendContact.ts

  

Responsibilities:

- Origin check
    
- Parse input
    
- Honeypot block
    
- Call service
    
- Handle errors
    

```
import { ActionError } from "astro:actions";
import { contactSchema } from "../contact.schema";
import { deliverContact } from "../service/deliverContact";
import { getAllowedOrigins } from "@/features/shared/config/allowed-origins";

export async function sendContactHandler(input: unknown, context: any) {
  // 1️⃣ Origin allowlist
  const allowedOrigins = getAllowedOrigins();

  if (allowedOrigins.size > 0) {
    const origin = context?.request.headers.get("origin");

    if (!origin || !allowedOrigins.has(origin)) {
      throw new ActionError({
        code: "FORBIDDEN",
        message: "Forbidden",
      });
    }
  }

  // 2️⃣ Validate
  const parsed = contactSchema.parse(input);

  // 3️⃣ Honeypot
  if (parsed.website.trim()) {
    return { success: true, message: "OK" };
  }

  // 4️⃣ Call service
  await deliverContact(parsed);

  return {
    success: true,
    message: "Your enquiry has been received.",
  };
}
```

Notice:

- No Resend logic here.
    
- No HTML.
    
- No formatting.
    
- Just orchestration.

# **5️⃣ Service Layer (Email Sending)**

  

**File:** service/deliverContact.ts

  

Responsibilities:

- Read env
    
- Escape user input
    
- Strip subject CRLF
    
- Send email
    
- Throw if config missing
    

```
import { Resend } from "resend";
import type { ContactFormData } from "../contact.schema";

function escapeHtml(str: string) {
  return str
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;");
}

export async function deliverContact(input: ContactFormData): Promise<void> {
  const apiKey = import.meta.env.RESEND_API_KEY;
  const toEmail = import.meta.env.CONTACT_TO_EMAIL;

  if (!apiKey) throw new Error("Missing RESEND_API_KEY");
  if (!toEmail) throw new Error("Missing CONTACT_TO_EMAIL");

  const resend = new Resend(apiKey);

  const safeName = escapeHtml(input.name);
  const safeEmail = escapeHtml(input.email);
  const safePhone = escapeHtml(input.phone);
  const safePractice = escapeHtml(input.practiceArea);
  const safeMessage = input.message
    ? escapeHtml(input.message).replace(/\n/g, "<br/>")
    : "—";

  const subject = `New enquiry – ${safePractice}`.replace(/[\r\n]+/g, " ");

  await resend.emails.send({
    from: "Mary Molloy Solicitor <info@yourdomain.com>",
    to: toEmail.split(","),
    replyTo: input.email,
    subject,
    html: `
      <h2>New Enquiry</h2>
      <p><strong>Name:</strong> ${safeName}</p>
      <p><strong>Email:</strong> ${safeEmail}</p>
      <p><strong>Phone:</strong> ${safePhone}</p>
      <p><strong>Practice Area:</strong> ${safePractice}</p>
      <p><strong>Message:</strong><br/>${safeMessage}</p>
    `,
  });
}
```

# **9️⃣ What You Must Test Manually**

  

### **Normal submission**

- Email arrives
    
- Reply goes to lead
    
- Formatting correct
    

  

### **Honeypot**

```
document.querySelector('[name="website"]').value = "spam"
```

Submit → should return success but not send email.

  

### **Origin**

  

Curl with wrong Origin → blocked.

  

### **Injection**

  

Submit <script> → email shows escaped text.
