## **SVGs in Astro: how to recolor them with Tailwind**

  

### **1) SVGs are text files**

  

An .svg isn’t “just an image” — it’s XML code.

  

To view/edit it:

- Right-click the SVG file in Cursor/VS Code
    
- **Open With → Text Editor**
    
- You’ll see <svg ...> ... </svg> markup
    

  

### **2) Make the SVG colorable**

  

If you want CSS/Tailwind to control the color, the SVG must use currentColor.

  

Change this:

```
<svg fill="#000000" ...>
```

To this:

```
<svg fill="currentColor" ...>
```

If it’s a stroke icon, you’ll see stroke="#000" — change it to:

```
stroke="currentColor"
```

### **3) Why** 

### **?raw**

###  **is required**

  

When you import normally, you’re not getting the SVG markup as a string.

  

?raw tells Astro/Vite: **import the file contents as a string** (the raw SVG code).

  

Example:

```
import companyLogo from "../assets/logo/companyLogo.svg?raw";
```

### **4) Why we used** 

### **set:html={companyLogo}**

  

Because companyLogo is now a string containing <svg>...</svg>.

  

Astro won’t render that string as markup by default — it will treat it like text.

  

set:html tells Astro:

- “Take this string and insert it as actual HTML.”
    

  

So:

```
<span set:html={companyLogo} />
```

means “render the SVG markup inside this span”.

  

### **5) Why we used a** 

### **<span>**

###  **instead of** 

### **<svg>**

  

Because we’re **injecting** an <svg> string. The wrapper element can be a span or div. It’s just a container for the injected SVG.

  

### **6) What this means:** 

### **[&_svg]:h-full [&_svg]:w-full [&_svg]:block**

  

That’s Tailwind’s “arbitrary selector” syntax. It styles the **child <svg> element inside the wrapper**.

- [&_svg]:h-full → set the nested <svg> height to 100%
    
- [&_svg]:w-full → set the nested <svg> width to 100%
    
- [&_svg]:block → make the nested <svg> display as block (avoids inline baseline gaps)
    

  

Why we need it:

- Many SVGs come with hardcoded width="800px" / height="800px".
    
- Your wrapper is h-8 w-8, but the SVG might ignore it.
    
- Forcing the inner SVG to h-full w-full makes it **fit the wrapper size** reliably.
    

  

### **7) How you actually set the color**

  

Because the SVG uses currentColor, you set color on the wrapper like any other element:

```
<span
  set:html={companyLogo}
  class="h-8 w-8 text-white [&_svg]:h-full [&_svg]:w-full [&_svg]:block"
/>
```

Swap text-white for your token:

- text-accent
    
- text-textOnDark
    
    etc.
    

---

If you want it even simpler, you can remove the selector stuff **only if** your SVG doesn’t have hardcoded width/height and behaves well — most don’t, so keep it.
